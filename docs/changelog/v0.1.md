# v0.1 — Core Trading System

**Status:** Released

## What Was Built

- Two-agent architecture: Analyst service (port 8001) + Executor service (port 8002), both FastAPI + Uvicorn
- Three-committee LLM framework — Fundamental Quality / Macro Regime / Technical Execution — running on Llama 3.1 8B via Ollama (fully local, no API cost)
- Two-stage audit pipeline: Analyst scores signal → if 70–75%, Executor independently stress-tests it → executes only if second score clears threshold
- Alpaca paper trading API integration
- Risk controls: 2% stop-loss per trade, 40% max position size, 3 trades/week (PDT compliance), 6% weekly loss kill switch
- Telegram bot with owner control commands and guest conversational mode
- Full-universe equity scan: S&P 500 + 300 liquid names via concurrent ThreadPoolExecutor
- tmux session management across windows: analyst, bot, executor
- Tailscale for remote SSH access
