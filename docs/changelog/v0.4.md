# v0.4 — Signal Pipeline Fixes + Broadcast Engine Rewrite

**Status:** Released

## What Was Built / Fixed

### Broadcast Engine
- **LLM removed from broadcast path**: rewrote `broadcaster.py` with pure technical scoring — RSI, MACD, EMA, Bollinger Bands, volume, momentum weighted scoring. Scan time dropped from 2–4 minutes to ~1.4 seconds.
- **yfinance MultiIndex fix**: newer yfinance returns `("Close", "AAPL")` tuple columns — flattened in `market.py`, `screener.py`, `multi_asset.py`, `analyzer.py`
- **MACD history fix**: period increased from 30d to 90d so MACD (requires 26+9 days minimum) always has enough data
- **Volume rolling mean fix**: `pd.NA` in volume column converts dtype to object and breaks `rolling().mean()` — fixed with `pd.to_numeric(...).replace(0, float("nan"))`
- **Tier 2 message split**: Tier 2 broadcast split into 5 messages (one per section) to stay within Telegram's 4096-character limit
- **Data timestamp**: broadcast headers now include `Data: X:XX PM ET`

### Signal Pipeline
- **`/testbroadcast` command**: owner-only on-demand broadcast trigger with elapsed time display
- **WATCH signals**: changed `SIGNAL_PROMPT` from returning `HOLD` to `WATCH` in neutral markets; WATCH signals now stored with calculated price targets
- **Confidence threshold**: lowered from 75% to 66%
- **Null target fallback**: if LLM omits `price_target`/`stop_loss`, calculated from price using standard R/R defaults (±4% stocks, ±3% forex)
- **SQLite permissions fix**: file permissions `644` block WAL journal creation — must be `664`

### Bot Commands
- **`/start`**: updated with full owner command list (7 categories) and separate guest menu
- **Guest rate limiting**: `/news`, `/predictions`, `/suggestions`, `/setups` rate-limited per user
- **`cmd_pause` fix**: was sending hardcoded string instead of actual result

## Breaking Changes
- Tier 2 channel format changed: now 5 messages per broadcast instead of 1
