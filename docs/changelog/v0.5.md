# v0.5 — 24/7 Scanning, Production Hardening, Social Integration

**Status:** Current

## What Was Built

### 24/7 Scan Engine
- **Argus never sleeps**: scan loop runs continuously, no market-hours gating on the scanner
  - Pre-market (07:00–09:30 ET): full stock universe + extended assets, every 30 min
  - Market hours (09:30–16:00 ET): core stock universe + extended, every 15 min
  - After-hours (16:00–20:00 ET): stocks + extended, every 30 min
  - Overnight (20:00–07:00 ET): crypto, forex, metals only, every 60 min
- **Extended asset scanning 24/7**: `run_extended_scan()` watches 23 crypto/forex/metals assets on every cycle — these markets never close
- **Dedup by recency**: tickers skip if scored within the last 4 hours (stocks) or 2 hours (extended assets), replacing the old once-per-day block

### Pure Technical Signal Pipeline
- **LLM removed from signal scan**: replaced `analyze_ticker()` (Ollama) with `score_snapshot()` (pure technicals). Scan time dropped from 10+ minutes to ~30 seconds.
- **Shared technical scorer** (`analyst/signals/technical.py`): single source of truth for scoring used by both the broadcast engine and the signal scanner
- **Parallel snapshot fetching**: `ThreadPoolExecutor(max_workers=8)` fetches all candidate snapshots simultaneously before scoring

### Social Intelligence Wired Into Scans
- Reddit, Bluesky, and Truth Social conviction scores now applied as confidence modifiers
- Cross-platform buzz (trending on 2+ platforms): +0.04 confidence when sentiment aligns with signal direction
- Single-platform: +0.02 / headwind: −0.02
- Social data fetched once per scan cycle (cached daily) — does not slow the scan
- Twitter/X integration already built; activates when `TWITTER_BEARER_TOKEN` is added to environment

### Production Hardening
- **Asset type isolation**: `asset_type` column added to `signals` table. Executor only processes `asset_type = 'stock'` — forex/metals/crypto signals are tracked but never routed to the stock brokerage
- **Double-entry guard**: executor checks open Alpaca positions before executing — prevents re-entering a ticker we already hold after a re-scan generates a second signal
- **UTC timezone correctness**: all `date.today()` references replaced with `datetime.now(timezone.utc).date()`. `generated_at` is stored in UTC; previous local-time comparisons caused signals to disappear after 8 PM ET
- **DB schema migration**: `_migrate()` runs on startup and applies `ALTER TABLE` changes to existing databases without data loss
- **Market hours guard on executor**: executor's signal watcher checks market hours before attempting any order — prevents after-hours order failures
- **Fractional stop-loss fix**: Alpaca rejects fractional quantity GTC orders — stop-loss quantity floored to whole shares; sub-1-share positions noted in logs
- **WATCH signal floor**: minimum confidence raised to 0.62 for all signal types — eliminates sub-threshold noise from the database
- **Weekly trade count uses ET**: Monday boundary for PDT week tracking now uses Eastern Time explicitly
