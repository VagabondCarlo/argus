# v0.6 — Production Bug Fix + Security Audit

**Status:** Current  
**Released:** 2026-06-17

## Summary

Full debug and pentest sweep of the live codebase. Thirteen issues found and patched across logic bugs, security vulnerabilities, and silent failure modes. System was running but dropping trades due to a chain of cascading failures — all identified and fixed.

---

## Bug Fixes

### Critical — Trade Execution Chain

**`analyst/data/universe.py` — S&P 500 fetch returning 37 tickers instead of 503**  
Wikipedia returns 403 Forbidden when `pd.read_html()` is called without a User-Agent. Fixed by adding `requests.get()` with a standard browser User-Agent and an in-memory process-level cache to avoid repeated fetches.

**`executor/main.py` — `get_open_positions` NameError on every watcher cycle**  
Function was used in `execute_signal()` but missing from the alpaca import line. Every signal watcher cycle crashed silently. Fixed by adding it to the import.

**`executor/gateway/alpaca.py` — wash trade rejection breaking trade records**  
Alpaca rejects a stop-loss order placed immediately after a market order (wash trade). The stop-loss failure propagated to the whole `place_order()` return value, which was treated as a trade failure — so trades were never written to the database. Fixed by splitting market order and stop-loss into separate `try/except` blocks with a 2-second sleep between them. Stop-loss failure is now a logged WARNING only; the trade itself still records.

**`executor/main.py` — signal watcher burning cycle on skipped signals**  
`process_pending_signals()` called `return` after any signal — whether executed or skipped. If MRVL was skipped (no position for a SELL), the watcher would sleep 5 minutes without checking the next signal in queue. Fixed: `execute_signal()` now returns `bool`, and the loop only returns on a real trade (`if traded: return`); skipped signals `continue` to the next one.

**`executor/main.py` — fractional short selling rejection**  
Executor tried to short stocks on SELL signals for tickers we don't own. Alpaca paper API rejects "fractional orders cannot be sold short." Fixed with 4-branch logic:
- BUY + existing position → skip (no double-entry)
- SELL + no position → skip (no shorting)
- SELL + existing → `close_position()`
- BUY + no position → `place_order()`

**`shared/database.py` — INNER JOIN hiding backfilled trades**  
`get_todays_trades()` and `get_trade_history()` used `INNER JOIN signals` — any trade with `signal_id = 0` (manual/backfilled) returned nothing. Fixed with `LEFT JOIN` + `COALESCE(s.ticker, t.order_id)` so all trades are visible regardless of whether a signal record exists.

### Logic Bugs

**`analyst/signals/scorer.py` — WATCH floor check never triggered**  
`technical.py` returns `"WATCH"` but scorer's floor check tested `action == "HOLD"` — a value that never appears. The 0.62 minimum floor for weak WATCH signals was silently bypassed for every signal. Fixed to check `action == "WATCH"`.

**`executor/audit/auditor.py` — weekly trade limit hardcoded to 3 in auditor**  
Audit hard block and display used the literal `3` regardless of `config.MAX_TRADES_PER_WEEK`. Now reads from config throughout.

**`executor/audit/auditor.py` — LLM prompt told to auto-veto at 3/3 trades**  
Prompt instruction "Weekly trades at 3/3 = auto veto" caused the LLM to block trades once 3 had been placed, even with the limit raised to 25. Replaced with a factual note that the hard rule block operates separately from the LLM verdict.

**`notifications/bot.py` — `/status` displayed "Trades today: X/3" hardcoded**  
Fixed to show weekly count from executor (`executor.get("trades_this_week")`) vs `config.MAX_TRADES_PER_WEEK`.

---

### Datetime Correctness

**`shared/database.py` — `__import__('datetime').timedelta()` hack**  
`timedelta` was missing from the import; a previous fix used an `__import__` workaround. Replaced with a proper top-level import.

**`shared/database.py` / `analyst/data/news.py` / `executor/risk/manager.py`**  
All remaining `datetime.utcnow()` calls replaced with `datetime.now(timezone.utc)`. Python deprecated `utcnow()` in 3.12.

**`shared/models.py` — `datetime.utcnow` in dataclass default factory**  
Fixed to `lambda: datetime.now(timezone.utc)`.

---

### Config Bug (Critical)

**`shared/config.py` — trading rule constants only defined when MASTER_KEY is unset**  
A merge error placed all trading rule constants (`CONFIDENCE_THRESHOLD`, `MAX_TRADES_PER_WEEK`, `MAX_POSITION_SIZE`, etc.) inside an `if not config.MASTER_KEY:` block, plus a duplicate `config = Config()` at module end. This meant every attribute access would raise `AttributeError` in production (where MASTER_KEY is set). Restructured so all constants are `Config` class attributes, unconditional.

## Security Fixes

**`notifications/bot.py` — master key comparison using `in` operator (timing attack)**  
`arg == config.MASTER_KEY` allows a timing side-channel. Replaced with `secrets.compare_digest()` across all key-checking paths.

**`notifications/bot.py` — startup crash on invalid `TELEGRAM_CHAT_ID`**  
`int(config.TELEGRAM_CHAT_ID)` with a `None` value raises `ValueError` at import time. Wrapped in `try/except`.

**`notifications/bot.py` — `send_sync_notification()` blocked event loop indefinitely**  
`requests.post()` with no timeout. Added `timeout=10` and `try/except` wrapper.

**`shared/config.py` — missing MASTER_KEY logged as critical on startup**  
Argus now logs a `CRITICAL` warning at boot if `MASTER_KEY` is absent, rather than silently accepting all destructive commands.

**`analyst/data/openclaw_client.py` — user input passed unsanitized to subprocess**  
Added `_sanitize_question()`: strips non-printable/non-ASCII characters and caps at 500 chars before passing to `subprocess.run()`.

**`analyst/data/browser_scraper.py` — SSRF blocklist bypass via hex/octal IPs**  
Expanded `_BLOCKED_HOSTS` to include GCP metadata (`metadata.google.internal`) and ECS task metadata (`169.254.170.2`). Added `_BLOCKED_IP_RE` to block hex-encoded IPs (`0xC0A80001`), octal IPs (`0300.168.0.1`), IPv6-mapped loopback (`::ffff:127.x.x.x`), and IPv6 link-local (`::ffff:169.254.x.x`).

---

## Trade Limit

Weekly limit raised from 3 → **25** (`MAX_TRADES_PER_WEEK=25`). The original 3-trade PDT rule was overly conservative for a paper account. 25 gives the system room to accumulate meaningful signal data across a full week.
